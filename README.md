# Генерация содержательного описания изображений открыток и плакатов

ВКР Магистратура ТГУ 2026
**Направление**: Компьютерное зрение и нейронные сети

## Обзор

Данный проект реализует прототип системы автоматической генерации структурированных и семантически насыщенных описаний изображений (открыток и плакатов). Основная цель - улучшение поиска и доступа к визуальным материалам в цифровых архивах и библиотеках.

Система объединяет методы компьютерного зрения и обработки естественного языка для преобразования визуального контента в читаемые человеком описания и поисковые метаданные.

---

## Архитектурный подход

Система предполагает реализацию в виде модульного vision-language пайплайна, разделяющего этапы восприятия, семантической интерпретации и генерации текста:

```
  - Генерация описания (пока - BLIP)
  - Семантическая классификация (пока - CLIP, zero-shot)
  - Сборка описания
  - Структурированный результат
```

Такой подход позволяет отказаться от жёстко заданных правил и использовать предобученные мультимодальные модели для обобщения на различные типы изображений.

## Используемые модели и режим работы

В проекте используются предобученные мультимодальные и языковые модели из библиотеки Hugging Face Transformers. Все модели загружаются и выполняются локально, без обращения к внешним API во время инференса.

## Модели

- **BLIP (Salesforce/blip-image-captioning-base)**  
  Используется для генерации текстового описания изображения (image captioning). Модель извлекает основные объекты и формирует базовое описание сцены на английском языке.

- **MarianMT (Helsinki-NLP/opus-mt-en-ru)**  
  Используется для перевода сгенерированного описания на русский язык. Обеспечивает формирование русскоязычных архивных описаний.

- **SigLIP (google/siglip-base-patch16-224)**  
  Используется для извлечения семантических признаков изображения в режиме zero-shot классификации. Модель сопоставляет изображение с набором текстовых гипотез и определяет:
  - тип изображения (открытка, плакат и др.)
  - стиль (винтажная иллюстрация, фотография и др.)
  - тематику (праздничная, пасхальная и др.)
  - эмоциональную окраску (настроение)

## Режим работы

Все модели работают в локальном режиме:

- при первом запуске веса моделей загружаются из Hugging Face Hub;
- после загрузки модели кэшируются на устройстве пользователя;
- дальнейшие вычисления выполняются полностью локально без сетевых запросов;
- поддерживается использование аппаратного ускорения (MPS на Apple Silicon, CUDA при наличии GPU).

Такой подход обеспечивает:
- автономность системы;
- воспроизводимость экспериментов;
- возможность использования в инфраструктуре библиотек и архивов без передачи данных во внешние сервисы.

## Предположительные основные компоненты
### 1. Caption Generator (BLIP)
- Модель: BLIP  
- Задача: генерация базового описания изображения  
- Выход:
  - описание на английском
  - перевод на русский  
- Примечание: используется как первичный источник фактов о сцене

### 2. Metadata Extractor (CLIP)
- Модель: CLIP (zero-shot)  
- Задача: извлечение семантических признаков через сравнение с предопределёнными классами  

- Извлекаемые категории:
  - тип изображения (postcard, poster, photo, illustration)
  - стиль (vintage, modern, watercolor, etc.)
  - тематика (holiday, romantic, historical, etc.)
  - настроение (happy, nostalgic, melancholic, etc.)

- Выход:
  - top-k метки по каждой категории + confidence

### 3. Theme Inference (Семантическая агрегация)
- Задача: нормализация и согласование признаков из CLIP  
- Вход:
  - raw-метаданные (CLIP)

- Выход:
```json
{
  "type": "...",
  "style": "...",
  "theme": "...",
  "mood": "..."
}
```
### 4. Description Builder (Генерация итогового описания)
Задача: объединение caption + semantic context
Вход:
caption (BLIP)
theme (Theme Inference)
Формирует:
развернутое архивное описание
поисковую строку (keywords / tags)
Особенности:
стабилизирует стиль текста
добавляет недостающий контекст (например, «винтажная открытка с праздничной тематикой»)

### 5. Evaluation Module (Оценка качества)
Задача: автоматическая оценка качества генерации
Лексические метрики:
BLEU
ROUGE
METEOR
Семантическая метрика:
BERTScore
Вход: 
- сгенерированные описания
- эталонные референсы (JSONL)

## Данные

Для реализации проекта необходимо собрать набор данных, содержащий открытки и плакаты, которые будут размечены каноническим описанием и полями для поиска. 
На основе этих размеченных данных будет осуществляться оценка качества работы системы. Также данные могут быть использованы для дообучения модели, если такие компоненты будут включены в финальную реализацию проекта.

## Примечания 
- В текущей версии используются предобученные модели, обучение не требуется.
- OCR (распознования текста) не включён в MVP из-за низкой надёжности извлечения текста через captioning.
- Архитектура спроектирована как расширяемая (возможна интеграция OCR, fine-tuning, специализированных классификаторов).
